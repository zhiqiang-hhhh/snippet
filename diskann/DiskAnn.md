# DiskANN 论文核心内容重组

论文：**DiskANN: Fast Accurate Billion-point Nearest Neighbor Search on a Single Node (NeurIPS 2019)**

## 1. 问题与目标

近似最近邻（ANN）在十亿级数据上通常面临三难：
- 高召回（high recall）
- 低延迟（few ms）
- 高密度（单机可承载更多点）

论文指出两类主流方案的局限：
- `IVF + PQ` 一类：内存占用低、速度快，但 `recall@1` 在高精度场景偏低（压缩有损）。
- 分片+多机内存图索引：召回可做高，但机器数和内存成本迅速上升。

DiskANN 的核心目标：**把图索引放到 SSD 上，同时保持高召回和低毫秒级延迟**。

## 2. Vamana：DiskANN 的图构建核心

DiskANN 的基础是新图算法 **Vamana**。它仍是“图上贪心搜索”范式，但通过更强的剪枝与参数化，得到更适合 SSD 的图结构（更小直径、可控度数）。

### 2.1 两个基础过程
- `GreedySearch(s, q, k, L)`：从起点 `s` 出发，在候选集大小 `L` 下做 best-first 扩展。
- `RobustPrune(p, V, α, R)`：对点 `p` 的候选邻居 `V` 做鲁棒剪枝，最终把出度控制在 `R`。

其中 `α` 是关键参数：
- 当 `α > 1`，剪枝更鼓励“长程边（long-range edges）”，有助于减少搜索 hop 数。
- 这对 SSD 场景尤为重要，因为 hop 数直接影响磁盘 round-trip 次数。

### 2.2 Vamana 构图流程（论文主干）
1. 初始化：随机 `R`-regular 有向图。
2. 选数据 medoid 作为全局起点 `s`。
3. 按随机顺序遍历所有点 `p`：
   - 先用 `GreedySearch(s, p, 1, L)` 找访问集 `V`；
   - 对 `p` 执行 `RobustPrune(p, V, α, R)`；
   - 加反向边并处理出度超限。
4. 做两遍：第一遍 `α=1`，第二遍 `α=用户设定值(>=1)`，实证上图质量更好。

### 2.3 与 HNSW/NSG 的关键差异
- Vamana 显式引入 `α`，可在“图度数-图直径”之间做更好权衡。
- 候选集使用访问过的节点集合（而非仅最终结果），更利于形成远程连接。
- 随机图初始化 + 双遍优化，构建质量和效率表现稳定。

## 3. DiskANN：SSD 常驻索引系统设计

### 3.1 大规模构建：重叠分区 + 合并图
为在有限内存下构建十亿级索引，论文采用：
1. `k-means` 切成 `k` 个簇；
2. 每个点分配到最近的 `l` 个簇（重叠分配，常用 `l=2`）；
3. 每簇独立构建 Vamana 子图；
4. 合并所有子图边集成总图。

好处：构建阶段内存可控；查询阶段仍在统一图上搜索，不必像“硬分片”那样跨多个独立索引聚合。

### 3.2 存储布局
- 内存（DRAM）：所有点的 PQ 压缩向量（用于快速近似距离）。
- SSD：每点的全精度向量 + 邻接表（固定长度块，便于按 ID 直接定位偏移）。

### 3.3 BeamSearch（降低 SSD 往返）
普通贪心搜索每次扩一节点，round-trip 多。DiskANN 改为每轮并行扩展 `W` 个前沿点：
- `W=1` 近似普通图搜索；
- 常用 `W=2/4/8`，在延迟、吞吐、带宽利用之间折中更好。

### 3.4 缓存与隐式重排
- 缓存策略：缓存起点附近若干 hop 的高频节点，减少随机读。
- 隐式重排：访问邻接时顺带读到该点全精度向量，最终用全精度距离重排 top-k；
  由于同扇区读取，这一步几乎不增加额外 I/O 次数。

## 4. 论文实验结论（最关键数字）

- 在 `SIFT1B` 上，DiskANN 在单机（16 核、64GB RAM + commodity SSD）实现：
  - `95%+` 的 `1-recall@1`
  - 毫秒级延迟（文中给出 `<5ms`，并报告可到 `<3ms` 均值）
  - QPS 级别在数千（文中报告 `>5000 QPS`）
- 对比压缩倒排路线（如 IVFOADC+G+P），DiskANN 在相近内存占用下可显著提高 `recall@1`。
- 对比 HNSW/NSG，Vamana 在 SSD 场景下通常需要更少 hop，更适合磁盘图搜索。

## 5. 参数与工程启发（落地版）

- `R`（最大出度）：增大通常提升召回、降低 hop，但会增索引体积和构建成本。
- `L`（搜索列表大小）：增大可提升召回，延迟也上升。
- `α`（剪枝强度）：`>1` 往往能得到更短路径图，SSD 查询受益明显。
- `W`（beam width）：过小 round-trip 多，过大浪费带宽与算力；`2~8` 常见。
- `PQ` 字节数：压缩更强省内存但近似误差变大，需要靠全精度重排兜底。

## 6. 一句话总结

**DiskANN 的本质是：用 Vamana 构造“低 hop 可导航图”，再把“PQ 内存近似 + SSD 全精度邻接/向量 + BeamSearch”组合成可在单机上支撑十亿级高召回 ANN 的系统。**
